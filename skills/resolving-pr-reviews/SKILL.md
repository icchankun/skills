# PRレビュースレッド対応

依存: `gh` CLI（認証済み）、`ruby`

## ワークフロー

このチェックリストをコピーして進捗を追跡する:

```
- [ ] ステップ1: 未解決スレッド取得
- [ ] ステップ2: スレッド一覧表示 → ユーザー選択
- [ ] ステップ3: 調査（Explore エージェント）
- [ ] ステップ4: 修正計画（Plan エージェント） → ユーザー承認
- [ ] ステップ5: 実装
- [ ] ステップ6: コミット → ユーザー確認
- [ ] ステップ7: プッシュ
- [ ] ステップ8: 返信
- [ ] ステップ9: 結果報告
```

## ステップ1: 未解決スレッドの取得

引数がない場合はPR番号を聞く。PRのブランチに切り替わっていない場合は最初に切り替える。

以下を正確に実行する:

```bash
REPO=$(gh repo view --json nameWithOwner --jq '.nameWithOwner')
ruby ~/.claude/skills/resolving-pr-reviews/scripts/fetch_pr_comments.rb "$REPO" <PR番号>
```

0件なら終了。

## ステップ2: スレッド一覧とユーザー選択

以下の形式で表示:

```
## 未解決レビュースレッド（N件）

### [1] path/to/file.rb:L42
> スレッド本文
by @reviewer (User) - 2024-01-15
```

- 各スレッドの最初のコメント（レビュアーの指摘）を表示。返信がある場合はやり取りも要約
- `author_type` を表示する（User / Bot）
- ユーザーに番号（`1`, `1,3,5`）または `all` で選択してもらう

## ステップ3: 調査（Explore エージェント）

選択された**全スレッド**の影響範囲を Agent ツール（subagent_type: Explore）で調査する。全スレッドの調査結果をまとめてユーザーに報告する。

## ステップ4: 修正計画（Plan エージェント）

**全スレッド**の修正計画を Agent ツール（subagent_type: Plan）でまとめて立てる。ステップ2の番号を使って各スレッドの対応方針を提示し、ユーザーの承認を得る。

ユーザーは番号を指定して個別に方針を修正できる（例: `2 はこうして`）。

## ステップ5: 実装

承認された計画に基づいて、全スレッドの修正をまとめて実装する。実装後、各スレッドの変更内容を番号付きで報告し、ユーザーに確認を得る。

ユーザーは番号を指定して個別に修正を依頼できる（例: `3 を直して`）。

## ステップ6: コミット → ユーザー確認

**新規コミット vs fixup の判断基準:**
- **fixup**: レビュー指摘に対する小さな修正（既存コミットの意図を変えない）
- **新規コミット**: 新しいロジック追加やリファクタリングなど、既存コミットとは異なる意図の変更

`/git-commit` スキルでコミットを作成する。

コミット後、差分をユーザーに提示し、プッシュの承認を得る。**自動でプッシュしない。**

## ステップ7: プッシュ

ユーザーの承認後、`git push` を実行する。

## ステップ8: 返信

対応した各スレッドに `gh api` で返信を投稿する。

コミットURLを含める場合は、`gh api` でGitHub上にコミットが存在することを確認してから返信する（リンク切れ防止）。

**返信テンプレート（「こちら対応しました！」+ 以下のいずれか）:**

- 新規コミット: `[<コミットメッセージの1行目>](<コミットのURL>)`
- fixup: `<変更箇所のコードの permanent link>`

**返信の投稿:**
```bash
gh api repos/{owner}/{repo}/pulls/{pr_number}/comments/{comment_id}/replies -f body="返信本文"
```

**resolve のルール（スレッドの最初のコメントの `author_type` で判定）:**
- `Bot` → 返信後に resolve する（GraphQL mutation `resolveReviewThread` を `thread_id` で実行）
- `User` → 返信のみ。resolve はレビュアーが行う

## ステップ9: 結果報告

```
## 対応結果

### [1] path/to/file.rb:L42 - 対応完了
- 変更内容: XXXを修正
- 変更ファイル: path/to/file.rb, spec/path/to/file_spec.rb
- 返信: 投稿済み / resolved
```

## 注意事項

- スレッドの意図が不明確な場合は推測せずユーザーに確認する
