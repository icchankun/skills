# /handover

セッション中の作業内容・意思決定・捨てた選択肢を、現在のworktreeの `.claude/HANDOVER.md` に保存する。

## ステップ1: 作業状態の収集

以下を**並列**で実行:

```bash
git branch --show-current
pwd
git status --short
git log --oneline -10
```

## ステップ2: セッション内容の整理

会話履歴から以下を整理する。「なぜそうしたか」の記録を重視すること:

1. **タスク概要**: 取り組んでいたタスク（チケットURL含む）
2. **完了した作業**: コミット単位で整理
3. **未完了の作業**: 残タスク、ファイルパス含む
4. **意思決定の記録**: 設計判断とその理由
5. **捨てた選択肢と理由**: 不採用アプローチとその根拠（同じ検討の繰り返しを防ぐ最重要情報）
6. **ハマりどころ・学び**: エラー、詰まったポイント、得た知見
7. **ブロッカー**: 進行を妨げている問題
8. **次のアクション**: 引き継ぎ先で最初にやるべきこと

## ステップ3: HANDOVER.md の作成

**保存先**: `<現在のworktreeパス>/.claude/HANDOVER.md`

```markdown
# Session Handover

- **作成日時**: YYYY-MM-DD HH:MM
- **トリガー**: manual | auto (PreCompact)
- **ブランチ**: <ブランチ名>
- **worktree**: <worktreeパス>

## タスク概要

<簡潔な説明。チケットURLがあれば記載>

## 完了した作業

- <作業内容。関連コミットSHAを添える>

## 未完了の作業

- <残タスク。ファイルパスや作業内容を含める>

## 意思決定の記録

- **<決定事項>**: <理由・背景>

## 捨てた選択肢と理由

- **<不採用アプローチ>**: <なぜ採用しなかったか>

## ハマりどころ・学び

- <エラーや詰まったポイント、得られた知見>

## ブロッカー・注意点

- <課題や注意すべきポイント>

## 現在の状態

### 未コミットの変更

<git status --short の出力。なければ「なし」>

### 直近のコミット

<git log --oneline の出力>

## 次のアクション

1. <最初にやるべきこと>
2. <次にやるべきこと>

## 関連ファイル

- <今回触った主要ファイルパス>
```

## ステップ4: 完了報告

- HANDOVER.md の保存パスを伝える
- `/takeover` で読み込めることを伝える
